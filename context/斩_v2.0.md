## 一、游戏概念

**堆叠消除 + 回合制构筑 + roguelike 爬塔——消除即是蓄力，出招即是斩杀。**

### 核心循环

爬塔 → 遭遇敌人 → 回合制对局：
① 从堆叠牌堆中点击选中可见牌
② 按顺序放入消除槽，点击顺序不可调整
③ 可提前结束回合，已放入的牌不可撤回
④ 自动结算：扫描槽内连续相同元素（≥3）→ 多段并行结算
⑤ 所有牌消耗 → 清空槽 → 进入下一回合
⑥ 循环直至一方血量为零
胜利 → 奖励选择
失败 → 本次爬塔结束

## 二、战斗系统（核心）

### 2.1 牌堆

- **牌库规模**：白盒原型 135 张（红 50 + 蓝 50 + 绿 35），用完即止
- 堆叠机制：3×3 展板，上层遮下层，只能点击当前可见牌
- 每摞牌厚度：15 张 → 3 层（3×3 表面可见，下层 3×3 部分遮挡）

### 2.2 消除槽（白盒原型 6 格 / 正式版 10 格）

牌放入后不可撤回，顺序从左到右固定。

#### 进槽操作（移动端优化）

**点选 + 确认吸入**：
1. 单指点击牌堆中的牌 → 牌飞出半格悬浮，消除槽对应位置显示**虚线预览位**
2. 再次点击（或点击消除槽中对应的虚线位）→ 牌吸入槽内
3. 替代双击，避免 iOS Safari 缩放冲突

#### 即时预览

消除槽上方显示**当前槽位连击预览**，随牌加入动态更新。

#### 结算规则

- 扫描槽内连续相同元素（必须相邻且同色），连续 ≥ 3 个 → 按连击表计算
- **多段并行结算**：同时识别所有连击组，各自计算效果
- **所有槽内牌（包括未合成的单牌）全部消耗掉**

#### 未消除惩罚

回合结束时，槽内**未形成连击的牌**每张扣玩家 **1 HP**。防止无脑过牌。

### 2.3 核心策略

| 策略 | 说明 |
|------|------|
| **看牌** | 点走上层牌后下层暴露。即使没合成，看清下层分布也是收益 |
| **垫刀** | 故意点走杂牌暴露下方好牌，顺便可能凑出连击 |
| **憋大招** | 布局几回合攒同色，打出高连击一波带走 |
| **空库流** | 牌库全消直接获胜 |

### 2.4 连击结算表（V7 实际数值）

**攻击/防御/治疗共用同一数值表：**

| 连击 | 数值 | 备注 |
|:---:|:---:|------|
| 3 | 3 | 基础，3连起效 |
| 4 | 5 | — |
| 5 | 8 | — |
| 6 | 12 | — |
| 7 | 16 | 爆发 |
| 8 | 19 | — |
| 9 | 22 | — |
| 10 | 25 | 满槽 |

- 攻击 = 此值伤害
- 防御 = 此值护盾
- 治疗 = 此值回血

**设计原则**：线性温和增长（3→25），避免指数爆炸导致平衡崩溃。在实战测试稳定后可按需调整。

## 三、敌人系统

> 防止玩家无限憋连击，必须有行动压力逼玩家做取舍。

### 3.1 敌人属性

| 属性 | 数值 |
|:----:|:----:|
| HP | 100 |
| 初始护盾 | 0 |
| 基础攻击 | 12 |
| 力量增长 | 每回合 +2 |
| 狂暴 | 每 5 回合攻击×2 |

### 3.2 玩家初始属性

| 属性 | 数值 |
|:----:|:----:|
| HP | 100 |
| 护盾 | 0 |

### 3.3 胜利/失败条件

- 胜利：牌库全消 或 敌人 HP ≤ 0
- 失败：玩家 HP ≤ 0

### 3.4 固定行为循环

| 回合 | 行为 |
|:---:|:----:|
| 1 | 攻击 |
| 2 | 防御 |
| 3 | +力量 |
| 4 | 攻击 |
| 5 | 攻击 |
| 6 | 防御 |
| 7 | **狂暴（攻击×2）** |
| (8+) | 回到回合 1 循环 |

固定循环便于测试平衡——不会因为 AI 随机性干扰对玩家策略的观察。

### 3.5 战斗流程

```
① 消牌阶段：观察牌堆 → 点选进槽 → 结束回合
   勇者行动开始 → 清空勇者旧盾
② 勇者结算：扫描连击 → 多段并行 → ①回血→②加盾→③攻击→未消除惩罚 → 所有牌消耗
   幽灵行动开始 → 清空幽灵旧盾
③ 幽灵行动：执行固定行动（攻击/防御/加力量）——此时勇者新盾有效，可挡伤
④ 回合结束 → 回合计数器+1 → 回到①
```

**护盾生命周期**：
- 勇者加的盾：勇者行动时加上 → 持续到**下回合勇者行动开始**（正好挡幽灵整轮攻击）
- 幽灵加的盾：幽灵行动时加上 → 持续到**下回合幽灵行动开始**（挡勇者整轮攻击）
- 护盾在**各自行动开始前清空**，不能跨回合堆叠
- 类似杀戮尖塔的护甲机制

### 7.3 奇遇事件（10种，随机触发）

| 概率 | 事件 | 效果 |
|:---:|------|------|
| 30% | 增删卡 | 三选一增卡5张 + 三选一删卡5张 |
| 30% | 强制塞卡 | 随机塞入牌库5张（普通类型之一，非诅咒）|
| 10% | 打小怪 | 战斗胜利后增删卡 + 10金币 |
| 10% | 打精英 | 战斗胜利后增删卡 + 20金币 + 遗物 |
| 10% | 回血 | 直接回复10点HP |
| 10% | 扣血 | 损失10点HP |

**奇遇塞卡的设计理念**：
- 塞入的是普通类型卡（攻击/防御/回血等），不是诅咒
- **不是纯负收益**——增加了牌库厚度，但稀释了build浓度
- 配合增删卡机制：奇数塞卡→偶数删卡，牌库厚度在"膨胀vs收缩"之间动态平衡
- 玩法含义：打薄build追求元气弹 vs 打厚build追求combo稳定性

### 7.4 休息站

决战前唯一休息点，**二选一**：

| 选择 | 效果 | 适用场景 |
|------|------|---------|
| 回血30 | 直接回复30HP | 血量危险、稳妥通关 |
| 增强牌组 | 选一种类型增强 | 血量充足、追求爆发 |

**增强牌组效果**：
- 该类牌连击效果 ×1.5（攻击/防御/回血）
- Buff/Debuff 门槛降低：`3→2, 5→4, 6→5, 7→6` 变成 `2→1, 4→2, 5→3, 6→4`
- 万能牌 → 一张顶两张

**设计意图**："没血但想拼一波"vs"稳一手回血"——玩家会记住这种高张力选择。
增强后"激动的心颤抖的手，一鼓作气斩BOSS"是核心爽点。

### 7.5 增删卡机制

**所有增删卡都是随机三选一，不一张张选**：

| 操作 | 方式 | 来源 |
|------|------|------|
| 增卡 5张 | 随机三色 → 选一种加入5张 | 普通怪/精英怪/商店 |
| 删卡 5张 | 随机三色 → 选一种删去5张 | 普通怪/精英怪/商店 |
| 大幅增卡 10张 | 随机三色 → 选一种加入10张 | 商店 |
| 大幅删卡 10张 | 随机三色 → 选一种删去10张 | 商店 |

**设计意图**：
- 不确定性驱动不同build方向（想走攻击流但随机出防御/回血/万能，必须重新思考）
- 增卡→该类连击概率↑，删卡→连击概率↓
- BOSS战核心矛盾：**薄牌库（离元气弹近但输出弱）vs 厚牌库（输出稳但离元气弹远）**
- BOSS设计需要让纯憋元气弹几乎不可能——单回合输出不够就会被压制

### 7.6 敌人难度曲线

普通怪 → 精英怪 → BOSS 的数值递增设计：

| 类型 | HP | 基础攻击 | 特点 |
|------|:--:|:-------:|------|
| 普通怪 | 100 | 12 | 7回合固定循环 |
| 精英怪 | 200 | 18 | 加新机制（塞牌/破盾/debuff）|
| BOSS | 待定 | 待定 | 多阶段，强力机制 |

## 附录 A：设计理念

1. **信息层级随机性** — 不是抽牌随机，是翻牌随机，信息暴露本身就是资源
2. **视野即是收益** — 哪怕空过，看清牌堆就是赚
3. **垫刀 = 资源管理** — 用杂牌换信息暴露
4. **爆发节奏** — 布局蓄力，一波斩杀
5. **你选的 Build 决定你怕什么** — 颜色干旱从运气变为选择
6. **永不刷新（大牌堆）** — 牌库比例就是你的策略
7. **元气弹终斩** — 空库不是苟活，而是蓄力一击

### 8.2 三关流程重构

```
第一关（教学关） → 第二关（核心体验） → 第三关（Boss 决战）
```

#### 第一关：教学关
- 目标：**必过，必爽**
- 牌堆：5×5 布局，仅75张（三色各25）
- 怪物：HP 50，基础攻击 6
- 敌人行动：仅攻击，无复杂循环
- 设计意图：类似《羊了个羊》第一关——让玩家以为"这游戏简单"，建立成就感

#### 第二关：核心体验关
- 使用当前原型逻辑（250张/10格/10种卡牌/7回合固定循环）
- 策略压力开始显现
- 失败可能开始出现

**第二关结束奖励**：
- 三选一圣物
- 额外机制：**看广告/分享 → 再获得一个圣物**

#### 第三关：Boss 决战
- 高难度卡点
- 高血量、高攻击、特殊机制
- 目标：制造"差一点通关"→ 触发广告/复活 → 翻盘爽感

---

### 8.3 战斗系统调整

#### 连击机制简化
- **取消复杂连击倍率**
- 三连即可生效（保留 COMBOS 表作为数值参考，但不做额外倍率叠加）
- 降低理解成本，提高直觉爽感

#### 结算顺序
```
Buff/Debuff 优先结算 → 攻击/防御计算
```
- Step1：攻击Buff、防御Buff、易伤、降攻等状态效果
- Step2：玩家攻击 → 玩家护盾 → 怪物攻击

#### 未消除惩罚保留
- 未形成连击的牌每张扣 1 HP
- 作用：保留压力感 → 制造残局 → 提供广告/分享转化点

---

### 8.4 IAA 广告设计

**核心原则：广告 = 玩家主动追求的资源**

禁止：强制打断、高频插屏、无意义广告

| 入口 | 触发时机 | 效果 | 形式 |
|------|---------|------|------|
| Boss后额外圣物 | 第二关结束 | 多获得一个圣物 | 激励广告 / 分享 |
| 圣物刷新 | 任意圣物选择界面 | 刷新当前圣物选项 | 分享优先 → 广告次选 |
| 洗牌 | 战斗中随时 | 重新打乱牌库 | 激励广告 |
| 消除槽转移 | 战斗中随时 | 槽内卡牌移至备用区 | 激励广告 |
| 半血复活 | 死亡时 | 半血继续战斗 | 激励广告 |

**分享换刷新的设计意图**：
- 分享场景：残局差一点通关 / Boss 差一点斩杀 / 拿到稀有圣物组合
- 用户心理："分享换刷新"比"看广告换刷新"更不烦
- 天然具备裂变传播属性

## 附录 A：设计理念

1. **信息层级随机性** — 不是抽牌随机，是翻牌随机，信息暴露本身就是资源
2. **视野即是收益** — 哪怕空过，看清牌堆就是赚
3. **垫刀 = 资源管理** — 用杂牌换信息暴露
4. **爆发节奏** — 布局蓄力，一波斩杀
5. **你选的 Build 决定你怕什么** — 颜色干旱从运气变为选择
6. **永不刷新（大牌堆）** — 牌库比例就是你的策略
7. **元气弹终斩** — 空库不是苟活，而是蓄力一击
8. **广告不是打扰，是逆转的代价** — IAA 嵌入玩法循环而非打断 (v2.0)
9. **分享是社交货币** — 残局截图 + 差一点通关 → 自然裂变 (v2.0)
