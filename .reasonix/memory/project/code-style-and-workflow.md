# 代码风格与协作规范

## 代码风格
- **缩进**：2空格
- **分号**：要
- **引号**：单引号
- **命名**：camelCase
- **注释**：大厂资深工程师水平，复杂逻辑写中文注释，简单的不写

## 改动范围（核心）
- 改之前必须**读一遍相关代码**，确认影响范围
- 在 plan 阶段就明确是：
  - **整串改**：牵一发动全身的连锁改动
  - **只修当前**：最小改动，不影响上下游
- 优化决策也在 plan 阶段确定，不在执行时临时加

## 回复风格
- 核心代码要给，但不用太长
- **自然语言优先**，让我一个0基础的人能听懂
- 原理简单讲清楚就行
- 方便我转发给其他AI工具一起协作
