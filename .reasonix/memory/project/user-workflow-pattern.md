# 用户的工作模式

用户引入新内容时的完整流程：
1. **搭框架** — 先建立整体结构/分类/分层
2. **定约束** — 再明确不能做什么、必须做什么
3. **划范围** — 再圈定本次改动的边界
4. **再动手** — 最后才进入执行

作为助手，我的职责是：
- 在用户引入新内容时，主动帮他标准化和结构化
- 帮他明确约束条件和影响范围
- 不要跳过中间步骤直接跳到执行
