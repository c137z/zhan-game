# 「斩」— IAA 小游戏项目书

> 回合制堆叠消除 × 轻策略构筑 | 纯前端 H5 | 微信/抖音小游戏
> 目标：短局高爽感 + 激励广告 + 分享裂变

---

## 一、项目概要

**一句话**：玩家从 5×5 堆叠牌堆中选牌放入消除槽，同色 3 连起效——攻击、叠盾、回血、上 Buff、挂 Debuff，与 Boss 回合制对战，直至一方倒下。

**灵感来源**：杀戮尖塔（构筑）+ 小丑牌（连击）+ 羊了个羊（堆叠消除）

**技术方案**：纯前端 HTML/CSS/JS，单页面，零后端，PWA 可安装，localStorage 存档。开发成本极低，一人可维护。

---

## 二、市场定位与差异化

### 2.1 赛道

IAA 小游戏（微信/抖音），轻策略 + 高爽感。不做深度 Roguelike，不追长线养成。

### 2.2 差异化：羊了个羊 + 真正的策略

| 维度 | 不做什么 | 做什么 |
|------|---------|--------|
| 类型 | 不追深度肉鸽 | 轻策略 + 高爽感 |
| 门槛 | 不搞纯消消乐 | 无序匹配降低门槛，保留决策空间 |
| 深度 | 不做系统膨胀 | 三个核心抉择：看牌 / 垫刀 / 憋大招 |
| 商业化 | 不强制打断 | 广告 = 逆转的代价，嵌入玩法循环 |
| 成长 | 不做局内重数值 | 局外外观收集为主 |

**核心护城河**：无序匹配降门槛，连击系统保深度。新手能玩、老手能秀。

## 三、核心玩法

### 3.1 单局流程

```
选关 → 进入战斗 → 回合制对战 → 胜利/失败 → 结算奖励 → 下一关
```

一局约 10-15 分钟。目前 3 关精简流程：教学关 → 核心体验关 → Boss 决战关。

### 3.2 战斗循环

```
① 观察牌堆 → 点选进槽（5×5 堆叠，上层遮下层）
② 槽满或主动结束回合
③ 结算：Buff/Debuff 优先 → 攻击/防御/治疗
④ 敌人行动（固定循环 + 特殊机制）
⑤ 回合结束，循环
```

### 3.3 牌组与连击

约 9 种卡牌类型：攻击、防御、治疗、万能、降攻、易伤、眩晕、攻击加成、减伤。

同类型 3 连起效。连击长度决定追击倍率——高连击 = 高爆发 = 爽感峰值。

未形成连击的散牌回合结束扣血，防止无脑过牌。

### 3.4 胜利条件

敌人 HP 归零，或牌库全部打空（触发元气弹斩杀）。

## 四、内容规模

### 4.1 Boss 设计

10 个 Boss，各具独特机制——锁牌堆、锁槽位、隐藏意图、涂牌面、塞废牌、随机弃牌、限时回合、先手攻击、晕玩家、舔 Buff。机制覆盖牌堆、槽位、信息可见性、回合顺序等维度。

### 4.2 圣物系统

10+ 个圣物（Roguelike 遗物），分为规则改变型、功能型、数值型三类。圣物组合产生 Build 差异化——连击流、乌龟流、垫刀流、速杀流。

### 4.3 模式

- **冒险模式**：50 关线性闯关，难度递增
- **猫猫迷宫**：随机 Boss 战
- **猫王塔**：层层挑战

### 4.4 局外系统

外观收集（卡牌皮肤、剑气特效、背景主题、称号）+ 猫毛货币商店（用广告或通关获取，购买便利性道具）。

现有 7 项猫毛商品：额外刷新、瞳力（预读意图）、卡牌统计、生命上限提升、回合护盾、复苏（低血回血）、圣物亲和。

## 五、商业化设计

### 5.1 广告入口

广告嵌入玩法循环，不打断：

- 双圣物选择 → 看广告全拿
- 战斗中洗牌 → 看广告重置牌库
- 清空消除槽 → 看广告转移卡牌
- 半血复活 → 看广告继续

### 5.2 分享裂变

分享换圣物刷新、残局"差一点"截图天然社交货币。高连击截图、稀有皮肤展示具备炫耀属性。

### 5.3 核心逻辑

> "差一点"张力：玩家可以接受自己菜，不能接受系统明显搞他。

## 六、画面与音效设计

### 6.1 美术风格

猫猫主题，Q 版 + 日系轻量风格。Emoji 作为卡牌和角色图标（低开发成本），后续可替换为定制美术资源。

### 6.2 连击爽感体系

**画面分层**：

| 层次 | 触发条件 | 视觉表现 |
|------|---------|---------|
| 基础攻击 | 3-4 连 | 小剑气划过，单段打击 |
| 追击攻击 | 5-6 连 | 多段追击动画，画面震动 |
| 大招爆发 | 7+ 连 | 全屏特效 + 伤害数字放大弹出 + Boss 受击反馈 |
| 元气弹斩杀 | 牌库清零 | 专属终斩动画，全屏闪光 |

- 伤害数字：低伤害白色 → 中伤害红色 → 高伤害金色大字体 + 描边
- Boss 受击：血量条震动 + Boss 图标抖动
- 连击数实时预览：槽位上方动态显示当前连击类型和预估数值

### 6.3 音效分层

| 层级 | 触发条件 | 音效方向 |
|------|---------|---------|
| 选牌 | 每次点选 | 清脆点击音 |
| 进槽 | 牌入消除槽 | 吸入/放置音 |
| 连击触发 | 3 连起效 | 渐强 rising tone |
| 追击 | 5 连以上 | 叠加二次音效 |
| Boss 受击 | 承受伤害 | Boss 专属吼声/受击音 |
| 回合切换 | 回合开始/结束 | UI 过渡音 |
| 局终 | 胜利/失败 | 主题旋律片段 |

**设计原则**：音效随连击数递增响度和层次——高连击 = 音效爆点 = 多巴胺峰值。

## 七、养成系统

### 7.1 设计原则

局外成长以**外观为主**，几乎不影响战斗数值。不做刷装备、不做重养成。

**为什么**：IAA 小游戏用户不预期在微信里"刷装备"。重数值养成 = 新老玩家断层。外观收集 = 炫耀 + 分享 = 裂变。

### 7.2 收集系统

| 系统 | 描述 | 解锁方式 |
|------|------|---------|
| 卡牌皮肤 | 改变卡牌图案/底色 | 通关次数、特殊成就 |
| 图鉴 | 记录遇到的卡牌、Boss、圣物 | 遇到即解锁 |
| 剑气特效 | 连击动画粒子/颜色 | 高连击次数 |
| 背景主题 | 战斗场景背景 | 击败特定 Boss |
| 称号 | 头衔展示 | 达成特殊条件 |

### 7.3 猫毛商店

通关/看广告获取猫毛货币。可购买：圣物刷新次数 +1、生命上限微提升、卡牌统计显示、回合护盾、复苏（低血回血）、圣物亲和等便利性道具。均为微量加成，不影响核心平衡。

## 八、新手引导

### 8.1 教学关设计

第一关专属设计：
- 简化牌组（仅攻击+防御），降低认知负荷
- 敌人弱（HP 低、攻击低、无特殊机制）
- 引导亮点：让玩家轻松打出 3-5 连 → 给炫酷反馈 → 建立"我懂了这个游戏"的决策自信
- ❌ 不是点三下就结束的无脑爽
- ✅ 是让玩家觉得自己做出了正确选择

### 8.2 渐进式系统解锁

- 教学关：仅攻击/防御两张卡
- 第 2-4 关：逐步引入 Buff/Debuff 牌
- 第 5 关起：Boss 开始有特殊机制
- 通关后解锁迷宫和猫王塔

### 8.3 信息提示

- 牌堆剩余数量实时显示
- 连击预览动态更新（槽位上方）
- Boss 意图预告（下回合做什么）
- 圣物效果文字说明

## 十一、风险与对策

| 风险 | 对策 |
|------|------|
| 玩法深度 vs 上手门槛失衡 | 无序匹配降门槛，连击追击保留深度，新手 3 连保底出伤 |
| IAA 广告打扰感 | 广告嵌入玩法循环（逆转资源），不做插屏打断 |
| 内容消耗速度 | 圣物组合 × Boss 机制交叉 = 高重玩性；每局随机种子不同 |
| 纯前端无后端约束 | 排行榜/社交功能通过平台 API 实现，核心玩法不需服务器 |
| 单开发者瓶颈 | 纯前端技术栈简单，内容配置化（新增 Boss/圣物仅需加配置） |
