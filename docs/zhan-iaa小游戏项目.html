《斩》项目评估稿

IAA 小游戏方向 · 行业交流

一、是什么

《斩》是一款"堆叠消除 × 回合制战斗 × 轻度构筑"的中轻度策略游戏。

玩家从 5×5 堆叠牌阵中选牌放入行动槽，同类型 ≥3 连自动结算，与 Boss 回合 制对战直至一方倒下。与三消的根本区别：玩家不是在找当下最优匹配，而是在为 未来几回合布局。

纯前端 H5，单页面，零后端。目标平台微信/抖音小游戏。1 人可独立开发与维护。

二、核心差异化

三个机制构成策略深度：

看牌——移除表层牌暴露下层信息。本回合低收益，可能是后续高价值的代价。

垫刀——主动消耗杂牌换视野和布局，短期损失换长期收益。

憋大招——多回合蓄力，高连击一波爆发，主要爽感来源。

连击引入追击倍率——连击段越长，伤害倍率越高，创造"憋长连"的策略激励。

三、产品结构

  • 主线 · 50 关冒险模式——三幕递进，教学和成长主路径
  • 日常 · 迷宫 + 爬塔——随机 Boss 战，日留存和长期目标
  • 10 个 Boss——猫咪主题，各具专属机制，同阶同强度
  • 10+ 圣物——局内获取，组合产生 Build 分化

四、爽感设计（核心竞争力）

数值、视觉、音效三层反馈同步升级：

  • 低收益（3-4 连）—— 轻量打击
  • 中收益（5-6 连）—— 剑气 + 屏震
  • 高收益（7+ 连）—— 多段斩击 + 大数字爆发

递进式音效与视觉同频，连击越高反馈越强。玩家能"看到自己布局的成果"。

五、养成与留存

局外成长定位"舒适度成长"——外观收集为主（皮肤、特效、主题、称号），附带 微量功能强化。不靠数值碾压破坏策略体验。

猫房系统（规划中）：击败的 Boss 猫进入猫房，每日撸猫获取货币——通过日常互 动提升次日和 7 日留存。

六、新手引导

进入即战斗，不设长文本教程。首关简化牌组 + 弱化敌人 → 玩家自然打出首次连击 → 产生"我懂了"的决策成就感。后续关卡逐步解锁新机制。

七、商业化

IAA 激励广告为主。广告出现在玩家主动触发场景：双圣物全拿、战斗中洗牌、战败 复活。嵌入玩法循环，与"差一点"情绪对齐。分享换刷新实现裂变。

八、技术方案

  • 平台：微信/抖音小游戏（H5）
  • 技术：纯前端 HTML+CSS+JS，单页面
  • 后端：无，localStorage 存档
  • 包体：约 3.5 MB
  • 团队：1 人可独立完成

九、开发状态

已完成：战斗引擎、10 Boss、10+ 圣物、50 关冒险、迷宫、猫王塔、猫毛 商店、手机适配、自动化测试、确定性随机系统。

短期重点：爽感落地、猫房系统、新手引导优化、外部玩家测试。
中期：平台接入、广告 SDK、排行榜。

预估：MVP 1~3 个月，Alpha 3~6 个月，商业验证约半年。

十、风险

  • 玩法深度 vs 门槛——3 连保底出伤 + 追击保留高手空间
  • IAA 打扰感——广告是逆转机会，不设强制插屏
  • 内容消耗——圣物 × Boss × 随机种子 = 每局策略不同
  • 无后端——核心玩法不需要，排行榜走平台 API

十一、希望获得的反馈

  1. 看牌/垫刀/憋大招是否构成足够差异化？
  2. 三层爽感设计能否支撑长期游玩？
  3. IAA 模式与品类的匹配度及结构性风险？
  4. 留存瓶颈最可能出现在哪个环节？
  5. 项目潜在市场量级区间？
