# 战斗日志面板改造方案

## 一、UI 位置

消除槽右上角，设置按钮旁边，加一个日志按钮`📋`（日志），不是之前的`⚔️`。

点击后弹出日志面板，固定在左上角（跟设置面板对称，设置面板在右上，日志面板在左上）。

右上角勾选框控制详简模式：
```
☐ 详情
```
默认不勾。

## 二、CSS 要求

- 面板最大高度 60vh，宽度 240px
- **隐藏滚动条**：`.log-panel::-webkit-scrollbar { display: none; }`，同时保留 `overflow-y: auto` 功能，手机触摸仍可滚动但看不到滚动条
- z-index 高于游戏主体，低于弹窗
- 底纹半透明毛玻璃效果，跟设置面板风格一致

## 三、精简模式

一行一个行为，只说结果不说过程。玩家行为在前，敌方行为在后。

```
—— 第 1 回合 ——
🗡 造成 10 点伤害
🛡 增加 8 点护盾
Boss 攻击造成 8 点伤害
—— 回合结束 ——
```

```
—— 第 2 回合 ——
❤️ 恢复 5 点生命
Boss 攻击造成 12 点伤害
—— 回合结束 ——
```

## 四、详情模式

右上角勾选`☑ 详情`后展开。

### 结构模板

```
—— 第 N 回合 ——
🃏 🗡🗡🗡🗡🗡🗡🗡🗡🗡🗟   ← 本回合出牌，最多 10 个 emoji，从左到右按打出顺序排列

📊 暴击×1.5  破甲×1.5       ← 玩家生效的 buff，没有就不写

🗡 攻击
   = 22(基础10连) ×1.5(暴击) ×1.5(破甲) - 5(护盾)
   = 45 点伤害
   Boss HP: 300 → 255  🛡️: 5 → 0

─── 敌方 ───
🐱 Boss 攻击
   = 12(基础) +4(力量) ×2(暴怒) ×0.7(减伤) ↓30%(虚弱)
   = 22 点伤害
   勇者 HP: 100 → 78  🛡️: 0

—— 回合结束 ——
```

### 颜色规范

| 因子 | 颜色 | 说明 |
|:----|:----|:-----|
| **基础值** | `#eee` 白色 | 公式起点，如 `22(基础10连)` |
| **×1.5(暴击)** | **`#f1c40f` 黄色** | 攻击加成 buff |
| **×1.5(破甲)** | **`#e74c3c` 红色** | 敌人受伤加深 debuff |
| **×0.7(减伤)** | **`#1abc9c` 青色** | 玩家减伤 buff |
| **↓30%(虚弱)** | **`#8e44ad` 紫色** | 敌人攻击降低 debuff |
| **-护盾** | **`#3498db` 蓝色** | 护盾吸收 |
| **- 护盾抵挡文字** | `#888` 灰色 | "被护盾抵挡" |
| **最终伤害** | **粗体 `#fff` 白色** | 结果大字 |
| **🃏 出牌行 emoji** | 每张卡自己的颜色 | 攻击红/防御蓝/治疗粉/破甲暗红等 |
| **📊 buff 行** | 每个文字按对应颜色 | |
| **─── 敌方 ───** | `#e74c3c` 红色分隔 | |
| **回合标题** | `#f1c40f` 黄色 | `—— 第 N 回合 ——` |
| **普通日志文字** | `#ccc` 浅灰 | `Boss 被击晕，跳过回合` |

### 数据结构

```js
// logLines 中每个条目
{
  side: 'player' | 'enemy',          // 区分我方还是敌方
  isTurnEnd: true,                    // 回合结束标记

  // 精简模式显示
  text: '🗡 造成 10 点伤害',

  // 详情模式展开
  cards: ['attack','attack','attack'],  // 本回合出牌类型数组
  buffs: [{ name: '暴击', value: '×1.5', color: '#f1c40f' }],  // 生效 buff
  formula: '4(基础3连) ×1.5(暴击)',     // 公式字符串
  formulaParts: [                       // 公式分段，每种颜色一段
    { text: '4(基础3连)', color: '#eee' },
    { text: ' ×1.5(暴击)', color: '#f1c40f' }
  ],
  finalValue: '10',                    // 最终数值
  detail: 'Boss HP: 300 → 290',        // 血量变化
}
```

## 五、隐藏滚动条

CSS 加一行：
```css
#log-panel::-webkit-scrollbar { display: none; }
#log-panel { -ms-overflow-style: none; scrollbar-width: none; }
```

## 六、按钮图标

战斗内消除槽右上角的日志按钮图标改为 `📋`，不是 `⚔️`（那个跟攻击卡牌图标冲突）。

## 七、实现思路

1. 在 `_resolveSlot` 结算完成后，收集本回合所有的出牌、buff、伤害数据
2. 组装成上述结构的条目，push 到 `st.logLines`
3. 渲染时根据 `☑ 详情` 勾选状态，选择精简版还是详细版渲染
4. 详细版的 `formulaParts` 用带颜色的 span 拼接
